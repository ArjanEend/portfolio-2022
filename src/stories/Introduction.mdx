# Neo-Brutalist Design System

Welcome to the official component catalog for **Arjan Cordia's Software Engineering & Platform Portfolio**.

This design system is built to show off frontend architecture practices, high contrast readability, and solid, framework-agnostic design principles.

---

## 🎨 Design Philosophy

Our design system is based on **Neo-Brutality**:
*   **High Contrast**: Pure colors, bold outlines, and heavy black drop-shadows.
*   **Responsive Typography**: Combining clean Sans-Serif (`Plus Jakarta Sans`) for reading and Display typeface (`Space Grotesk`) for headings.
*   **Accessibility First**: Enforced WCAG AA minimum contrast ratio and keyboard navigability with robust visual indicators.

---

## 🏗️ Core Components

Below is the list of available design system components:

1.  **Button**: Standardized click targets supporting primary (`duck-beak`), secondary (`duck-blue`), accent (`duck-yellow`), and round `icon` states.
2.  **Badge**: Versatile badge colors (`yellow`, `blue`, `coral`, `purple`, `green`, `white`) that also support absolute positioning and tilts when used as sticker labels.
3.  **Card**: Solid container layouts featuring hover elevation, offset transforms, and brutal shadows.
4.  **Heading**: Hierarchy heading nodes with options for underline waves and bottom outline rules.
5.  **InputField**: Accessible input wrappers that automatically connect labels and input nodes.
6.  **Avatar**: Brutalist profile framing animations.

---

## ♿ WCAG Compliance Notes

*   **Color Contrast**: We ensure a contrast of at least **4.5:1** on all text elements (such as using `#1A1A1A` on `#FF5D46`).
*   **Keyboard Navigation**: Interactive controls use clear focus outlines: `focus-visible:ring-4 focus-visible:ring-black focus-visible:ring-offset-2`.
*   **Screen Readers**: Form inputs are explicitly mapped with labels and `aria-required` tags.
